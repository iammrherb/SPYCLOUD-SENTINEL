Prerequisites

Ensure you have the following before deploying.

  • ✓
    SpyCloud Enterprise API Key - Obtain from portal.spycloud.com. Enterprise tier required; Enterprise+ unlocks Compass, IDLink, and Investigations APIs.
  • ✓
    Azure Subscription - With a Log Analytics workspace and Microsoft Sentinel enabled (or the template can create both for you).
  • ✓
    RBAC Permissions - Microsoft Sentinel Contributor, Log Analytics Contributor, and Logic App Contributor roles on the target resource group.
  • ✓
    Network Access - Outbound HTTPS (443) to api.spycloud.io from the Azure data collection infrastructure.
  • ✓
    Post-Deployment - Run the provided permissions script to grant managed identity API permissions to each playbook.

Quick Start Guide

Get up and running in four steps.

1

Deploy the Solution

Click "Deploy to Azure" and follow the guided wizard. Select your workspace, configure API settings, and choose which components to deploy.

2

Configure Permissions

After deployment, run scripts/grant-permissions.sh in Azure Cloud Shell to grant each playbook's managed identity the required API permissions.

3

Enable Analytics Rules

Navigate to Sentinel > Analytics > Rule Templates. Review SpyCloud rule templates and enable the ones appropriate for your environment.

4

Verify Data Flow

Open the SpyCloud Connector Health workbook to verify data ingestion. Initial data should appear within 15 minutes of deployment.
