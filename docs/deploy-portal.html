Deployment Options

Choose Your Path

Multiple ways to deploy — pick whatever fits your workflow.

ℹ The recommended approach for most users. The custom UI definition provides a guided wizard with validation, tooltips, and sensible defaults. No CLI or infrastructure-as-code experience required. 
# Step 1: Click "Deploy to Azure" above
# Step 2: Fill in the guided form:
#   - Select your Subscription & Resource Group
#   - Enter your SpyCloud API Key
#   - Choose workspace name (new or existing)
#   - Review + Create

# Step 3: After deployment, run post-deploy script:
chmod +x scripts/post-deploy.sh
./scripts/post-deploy.sh -g your-resource-group -w your-workspace-name
ℹ Use Azure Cloud Shell if you prefer the CLI or need to deploy from a CI/CD pipeline. Works in any browser with nothing to install. 
# Clone the repository
git clone https://github.com/iammrherb/SPYCLOUD-SENTINEL.git
cd SPYCLOUD-SENTINEL

# Create resource group (if needed)
az group create --name spycloud-sentinel --location eastus

# Deploy the ARM template
az deployment group create \
  --resource-group spycloud-sentinel \
  --template-file azuredeploy.json \
  --parameters azuredeploy.parameters.json \
  --parameters workspace=spycloud-ws spycloudApiKey=your-key

# Run post-deploy automation
chmod +x scripts/post-deploy.sh
./scripts/post-deploy.sh -g spycloud-sentinel -w spycloud-ws
ℹ For teams using Terraform for infrastructure management. The Terraform module wraps the ARM template with HCL variables and outputs. 
# Initialize and deploy with Terraform
cd terraform

# Configure your variables
cp terraform.tfvars.example terraform.tfvars
vim terraform.tfvars  # Set your API key, workspace, etc.

# Deploy
terraform init
terraform plan -out=tfplan
terraform apply tfplan

# Run post-deploy automation
cd ..
./scripts/post-deploy.sh -g $(terraform -chdir=terraform output -raw resource_group) \
  -w $(terraform -chdir=terraform output -raw workspace_name)
ℹ Automate deployments through GitHub Actions. Fork the repo, set secrets, and push to main to trigger a deployment. 
# 1. Fork the repository on GitHub

# 2. Add repository secrets:
#    AZURE_CREDENTIALS     - Service Principal JSON
#    SPYCLOUD_API_KEY      - Your SpyCloud API key
#    AZURE_RESOURCE_GROUP  - Target resource group
#    AZURE_WORKSPACE       - Sentinel workspace name

# 3. Push to main to trigger deployment:
git push origin main

# The workflow will:
#   ✓ Validate the ARM template
#   ✓ Deploy all resources
#   ✓ Run post-deployment configuration
#   ✓ Verify the deployment

FAQ

Frequently Asked Questions

What Azure permissions do I need? ▼
You need Contributor and Security Admin roles on the target subscription. The Contributor role allows resource deployment, and Security Admin is required for Sentinel analytics rules, automation rules, and playbook connections. If you plan to use the device isolation playbook, you also need MDE admin access.
Can I deploy into an existing Sentinel workspace? ▼
Yes. During the deployment wizard, set "Create New Workspace" to false and enter the exact name of your existing Log Analytics workspace. Make sure you select the same region where your workspace is deployed. All SpyCloud resources will be added alongside your existing Sentinel configuration.
Is the deployment idempotent? Can I re-run it? ▼
Yes. The ARM template uses incremental deployment mode. Re-running the deployment will update existing resources to match the template without duplicating anything. This is safe to run multiple times, for example after updating to a newer version.
What does the post-deploy script do that ARM cannot? ▼
ARM templates cannot perform certain operations: resolving the DCE Logs Ingestion URI, obtaining the DCR Immutable ID, assigning RBAC roles on the DCR itself, configuring Microsoft Graph API permissions, or granting admin consent. The post-deploy script handles all seven of these phases automatically.
How much does this cost to run? ▼
The primary cost is Log Analytics data ingestion, which depends on the volume of SpyCloud breach data pulled by your connectors. The playbooks (Logic Apps) run on a consumption plan and cost fractions of a cent per execution. Analytics rules and detection are included in your existing Sentinel pricing tier. Expect minimal incremental cost for most organizations.
Can I selectively deploy only certain components? ▼
The unified ARM template deploys everything together for the simplest experience. However, the deployment wizard includes toggles to enable or disable specific component groups (e.g., skip playbooks, skip hunting queries). For fine-grained control, use the Terraform module or deploy individual component JSON files from the repository.
Where do I get a SpyCloud API key? ▼
Contact SpyCloud to set up an account and obtain API credentials. You will need an API key with access to the Breach Catalog, Compass, and Enterprise Protection APIs depending on which connectors you enable.
